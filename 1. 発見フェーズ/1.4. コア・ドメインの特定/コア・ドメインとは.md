コアドメインとは、そのビジネスにとって最も重要で、競争優位性の源泉となる中心的な領域を指します。

### 分類と分析
ドメインには、３つの種類がある。

#### コアドメイン

```
コアドメインとは、複雑なロジックを持ち代替えできないドメイン領域を指します。
競合に勝つための門外不出の武器。

代替えの例）
代替えが可能： 
ID&PWを使った認証機能は、代替えが可能であるケースが多いです。
ID&PWを使った認証機能は、自身で制作せずとも0Authなど他サービスで代替え可能です。

代替えができない：
サービス特有のロジック、業界の中でも特殊な工程、社内ルールのようなロジックです。
他ではないためゼロから作らなければならない。
外注せず自身の会社で内製化することが望ましいドメイン領域です。
```


---

#### サブドメイン

```
サブドメインとは、開発する必要はあるのだけど複雑性のないロジックを持つ領域を指します。

サブドメインの特徴：
ETL（抽出、変換、格納）や CRUD（作成、読込、更新、削除）のような基本的なロジックがもとになっており複雑性はないものの自社固有の業務領域であるもの。
```


---

#### ジェネラルドメイン（汎用ドメイン）

```
ジェネラルドメインとは、複雑性があるものの外部で代用することが可能な領域です。

ジェネラルドメインの特徴：
丸ごとOAuthやStripeなどの外部リソースに置き換えれる領域であるもの。
例としては、決済 → Stripe, 認証 → 0Auth があります。
外部リソースなのでOSSとかでもあり。
```
