
・ドメインの状態を変化させ、次の処理をトリガーする事実

```
文脈にコマンド（=〜する。）がある時に、前後にイベントが存在しないか？

イベント　　　　　→　コマンド
商品が購入された　→　商品の在庫を減らす
			   →　決済を開始する
			   →　出庫準備をする
			   
コマンド　　　　　→　イベント
出庫準備する　　　→　出庫した
```


・過去完了系である

```
〜された。　のような単純に過去完了系はないか？

ただし、UI操作やデータの参照は除外する（顧客がカートを見た）
```


・業務の節目になる事実である

```
分岐・プロセスの完了を表す過去完了系は、ないか？

決済が完了した
決済が失敗した
```
